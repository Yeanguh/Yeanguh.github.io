mysql视图,事物和索引的使用

Sep 20, 2017 | Mysql

视图

  • 定义视图

    create view 视图名称 as select语句;

  • 查看视图

    show tables;

  • 使用视图

    select * from 视图名;

  • 删除视图

    drop view 视图名称;

视图特点

  • 视图封装了对多张基本表的复杂操作，简化用户操作
  • 视图只是一个虚表,并不存储任何基本表的表数据,当用户使用视图的时候 视图会从基本表中取出
  • 通过视图可以对用户展示指定字段从而屏蔽其他字段数据，更加安全

事物

定义

事务Transaction，是指作为一个基本工作单元执行的一系列SQL语句的操作，要么完全地执行，要么完全地都不执行。

事务四大特性ACID

原子性(Atomicity)

  • 强调事务中的多个操作时一个整体

一致性(Consistency)

  • 强调数据库中不会保存不一致状态

隔离性(Isolation)

  • 强调数据库中事务之间相互不可见

持久性(Durability)

  • 强调数据库能永久保存数据，一旦提交就不可撤销

事务的使用

开启事务

  • 开启事务后执行修改命令，变更会维护到本地缓存中，而不维护到物理表中

提交事务

  • 将缓存中的数据变更维护到物理表中

查看:

修改：

  • 可以关闭自动提交 就可以在自动开启事务的情况下 手动提交事务

回滚事务

  • 放弃缓存中变更的数据 表示事务执行失败 应该回到开始事务前的状态

坑点

  • 不能回滚的操作: 创建/删除 数据库/表结构 create/drop table等<隐式提交>

  • 能够回滚的操作 对表数据的 insert update delete

  • 自动触发事物开启和提交

注意:

mysql命令行 insert update delete默认自动开启事务 默认会自动提交

索引

索引的定义

索引是一种特殊的文件,保存着对数据表里所有记录的位置信息。例如书中的目录

在数据库中读取操作大于写入操作的比例,虽然使用索引在写入数时,会增加数据库的工作量,但是由于数据库的使用读取操作的占比较高,所以使用索引在整体效果上仍然是提高效率

索引的优缺点

优点

  • 极大的提高某一字段的查询效率

缺点

  • 建立太多的索引将会影响更新和插入的速度，因为它需要同样更新每个索引文件
  • 不适用于经常需要更新和插入的表格
  • 建立索引会占用磁盘空间
