常见的可用于版本控制的工具

Jul 5, 2018 | Git

全球最大的同性交友网站Github

Git的优缺点如下：

1．适合分布式开发，每一个个体都可以作为服务器。每一次Clone就是从服务器上pull到了所有的内容，包括版本信息。

2．公共服务器压力和数据量都不会太大。

3．速度快、灵活，分支之间可以任意切换。

任意两个开发者之间可以很容易的解决冲突，并且单机上就可以进行分支合并。

5．离线工作，不影响本地代码编写，等有网络连接以后可以再上传代码，并且在本地可以根据不同的需要，本地新建自己的分支。

中国的码云Gitee，此码云非彼马云

Gitee的优缺点如下：

Gitee：不和你多BB，github啥样我啥样，照搬全抄，但我是中文的微笑

集中式的版本管理系统SVN

优点：

管理方便，逻辑明确，操作简单，上手快。

易于管理，集中式服务器更能保证安全性。

代码一致性非常高。

有良好的目录级权限控制系统。

劣势:

对服务器性能要求高，数据库容量经常暴增，体量大。

必须联网。如果不能连接到服务器上，基本上不可以工作，如果服务器不能连接上，就不能提交，还原，对比等等。

不适合开源开发。

分支的管控方式不灵活

古董级别的VSS.aspx

VSS作为一款历史悠久的版本管理工具，在早期扛起了版本管理系统方面的大气，能帮助解决一部分版本控制方面的问题，但是依旧存在一些不足，比如：

文件大多会以独占的形势进行锁定。如果一个人在修改的时候其他人没有办法进行修改。

VSS只支持Windows版本，且只兼容微软的开发工具。

文件存储，服务器必须共享文件夹，对文件的安全性没有足够保障。

这里介绍Git的使用方法

  1. 首先注册github或者gitee账号

  2. 新建仓库的两种方式

    2.1 远程创建仓库克隆到本地 git clone 你要克隆的git仓库地址

    2.2 新建文件夹，文件夹下初始化仓库 cd 文件夹 | git init - -> 当前文件夹下生成 .git 文件

  1. 配置个人信息

    默认不配置的话，会使用全局配置里面的用户名和邮箱 全局git配置文件路径：~/.gitconfig

    3.1 设置用户名

    git config user.name ‘名字’

    3.2 设置用户邮箱

    git config user.email ‘邮箱账号’

  2. 生成秘钥和公钥

    4.1 把~目录下.ssh文件删除

    rm -rf .ssh

    4.2 重新生成自己的 .ssh文件

    ssh-keygen -t rsa -C ‘注册gitee时的邮箱’
cat id_rsa.pub

  3. 将文件所有内容复制到SSH添加，在远程仓库setting中的SSH设置处添加公钥

到此git就可以正常使用了

提交按照以下步骤

分支

分支和分支之间代码是不同, 相互隔离的

个人开发：默认使用master一个分支即可

多人开发：

master 分支为锁版代码存储分支

一般由项目负责人把dev分支的代码提交到master分支,再由运维或者负责人部署代码

dev分支

开发阶段性代码和代码测试解决bug
自己定义的普通分支，作为开发人员提交的分支

分支相关指令

查看所有分支

git branch

创建分支

git branch dev(分支名)

切换到分支

git checkout dev(分支名)

合并上面两个步骤

git checkout -b 名称

创建dev分支

git checkout -b dev origin/dev
服务器创建的dev分支和本地创建的dev,两个文件完全相同

合并分支

git merge 
直接覆盖源代码,不提示

当多人开发提交发生冲突

当同分支merge代码时先提示pull下来，如果没有冲突会正常pull下来

如果提示merge失败，就代表发生冲突, git会有提示请先处理本地的冲突

git stash的其他指令

git stash drop 
或者使用git stash clear命令，删除所有缓存的stash；

git stash show
在该命令后面添加-p或--patch可以查看特定stash的全部diff
